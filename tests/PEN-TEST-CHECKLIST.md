# Pen-test чеклист (ручное и полуавтоматическое)

## Автоматизируемые (локально)
1. `npm audit` — известные уязвимости зависимостей  
2. `npm run test` — регрессия логики (после `npm install`)  
3. `npm run build` — сборка без ошибок

## Поведение API (ручное, в DevTools / curl)
- [ ] `GET /api/projects` без cookie сессии → **401** (если реализован GET) — сейчас маршруты только POST/PATCH  
- [ ] `POST /api/projects` без cookie → **401**  
- [ ] `PATCH /api/projects/:id` с чужим `id` (другая организация) → **404** / отсутствие данных  
- [ ] `POST /api/projects` под VIEWER → **403**

## Сессии и заголовки
- [ ] Cookie `next-auth` — `HttpOnly` (проверка в Application → Cookies)  
- [ ] Нет секретов в ответах JSON  

## Инъекции
- [ ] Поля ввода: длинные строки / спецсимволы в названии проекта — не ломают UI (Zod ограничивает длину)
