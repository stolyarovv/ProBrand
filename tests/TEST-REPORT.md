## 4. Security review (код)

| Тема | Хорошо | Риски / улучшения |
|------|--------|-------------------|
| Аутентификация | NextAuth + credentials, сессия на сервере | Нет 2FA, нет блокировки после N попыток |
| Авторизация | Роль в JWT, проверка в API, `canAccessSection` в middleware | Нет rate limiting на `/api/*` |
| Данные | Prisma, параметризованные запросы | — |
| Валидация входа | Zod на POST/PATCH проектов | Расширить на остальные API |
| Секреты | `NEXTAUTH_SECRET`, `DATABASE_URL` в `.env` | Не коммитить `.env` в git |
| Заголовки | Next.js по умолчанию | Рассмотреть CSP, HSTS на проде |
